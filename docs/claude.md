# AI Engine - Project Instructions

> **Note:** Superseded by [CONTRIBUTING.md](../CONTRIBUTING.md) and [ARCHITECTURE.md](ARCHITECTURE.md).

## Project Overview

AI Engine v3.0 is a Python-based multi-provider AI gateway that provides:
- OpenAI-compatible API endpoints
- Support for 22+ AI providers with intelligent key rotation
- Web dashboard for monitoring and management
- Chat module with persistence and WebSocket support
- Autodecide feature for automatic provider selection

## Tech Stack

- **Backend:** Python 3.x, FastAPI, Uvicorn
- **Database:** SQLite (chat persistence), JSON files (statistics, cache)
- **Frontend:** Jinja2 templates, vanilla JavaScript
- **Key Libraries:** requests, aiohttp, pydantic, python-dotenv

## Project Structure

```
AI_engine/
├── ai_engine.py          # Core AI engine with provider management
├── server.py             # FastAPI server with API endpoints
├── config.py             # Provider configurations and settings
├── model_cache.py        # Model caching system
├── statistics_manager.py # Persistent statistics tracking
├── chat_module/          # Chat functionality
│   ├── router.py         # FastAPI routes for chat
│   ├── db.py             # SQLite database operations
│   └── websocket_manager.py  # WebSocket connection management
├── static/               # Static assets (JS, CSS)
├── templates/            # Jinja2 HTML templates
├── .agent/               # Agent-generated files (reviews, plans)
└── claude.md             # This file
```

## Development Guidelines

### Code Style
- Follow PEP 8 for Python code
- Use type hints for all function signatures
- Keep functions focused and under 50 lines when possible
- Use meaningful variable and function names

### Git Workflow
- Commit messages should be concise and descriptive
- Use conventional commits format: `type(scope): description`
- Types: feat, fix, docs, style, refactor, test, chore
- Example: `fix(engine): resolve duplicate initialization in AI_engine`

### Testing
- Run tests before committing: `pytest tests/`
- Aim for >80% code coverage on new features
- Test edge cases and error conditions

### API Conventions
- All API endpoints should follow RESTful patterns
- Use proper HTTP status codes
- Return consistent response formats
- Include error details in responses

## Common Tasks

### Adding a New Provider
1. Add provider configuration to `config.py` in `AI_CONFIGS`
2. If custom format needed, add handler in `ai_engine.py`
3. Test with `/api/test-model` endpoint
4. Update documentation

### Running the Server
```bash
# Install dependencies
pip install -r requirements.txt

# Start server
python server.py

# Or with uvicorn directly
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```

### Accessing Services
- Dashboard: http://localhost:8000
- API Docs: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## Environment Variables

All API keys should be stored in `.env` file (never commit):
```
A4F_API_KEY=your_key_here
CHI_API_KEY=your_key_here
# ... etc
```

## Security Notes

- Never commit `.env` files or API keys
- Use environment variables for all secrets
- Validate all user input at API boundaries
- Rate limit public-facing endpoints in production

## Debugging

### Enable Verbose Mode
Set `verbose_mode: True` in `ENGINE_SETTINGS` in `config.py` or use:
```python
engine.set_verbose(True)
```

### Check Statistics
```bash
curl http://localhost:8000/api/statistics
```

### Check Provider Status
```bash
curl http://localhost:8000/api/status
```

## Agent Files

The `.agent/` directory contains:
- `reviews/` - Code review documents
- `plans/` - Development roadmap and plans

These are generated by the AI assistant and should not be manually edited.
